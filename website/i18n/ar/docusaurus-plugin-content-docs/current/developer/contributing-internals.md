# تفاصيل المساهمة الداخلية

هذه الصفحة ستكون النسخة العربية لمراجعة الصفحة الإنجليزية المرتبطة.

## الغرض

سياقات المساهمة.

## ملاحظات مهمة

- اسماء الأوامر، الاختيارات، والمعاملات التقنية تبقى باللغة الإنجليزية.
- مسارات الملفات، ومتغيرات البيئة، ومعرفات المزودين تبقى باللغة الإنجليزية.
- لا تزال الترجمة الآلية الكاملة في هذه المرحلة.

## مصدر الحقيقة

مجموعة وثائق Docusaurus هذه هي المصدر العام لمطالبات v0.1.0 الموجهة للمستخدم. إذا خالفت السلوك المنفذ، يجب إصلاح الوثائق أو الكود قبل الشحن.

## قواعد تصيير `readline` والطرفية

عندما يملك `readline` المؤشر، لا تكتب مخرجات طرفية مباشرة حوله. صف المحث ليس سطر إخراج حر؛ قد يعيد `readline` رسمه بعد كل ضغطة مفتاح، أو لصق، أو تغيير عرض، أو إرسال.

استخدم مسار bottom chrome الواعي بـ `readline`:

```typescript
updateManagedRegionAboveReadline({ state, transientLines, promptLineCount })
```

قواعد التشغيل:

- لا تخلط يدويًا بين `updateTransientLines()` و `updateStateAboveReadline()` لتحديثات واجهة مباشرة بجوار المحث.
- احسب نمو وانكماش عدد أسطر المنطقة المُدارة.
- امسح الأسطر القديمة عندما تختفي تلميحات slash أو معاينة اللصق أو الرسائل المؤقتة.
- عامل عرض الطرفية كقيمة متغيرة؛ التفاف المحث قد يتغير أثناء تحرير المستخدم.
- لا تعكس محتوى مطالبة سرية في paste preview أو transient lines أو status rails أو السجلات أو debug chrome.
- أبقِ command lane أثناء الدور النشط داخل مسارات transient/status في bottom chrome. لا تكتب buffer الأمر مباشرة إلى الطرفية.
- لا تُعد صندوق محث وهمي بعد الإرسال. النص الذي أرسله المستخدم يخص transcript/history، أما active-turn chrome فهو للحالة والتحكم.

أي تغيير في cursor control يحتاج اختبار وحدة وفحصًا يدويًا في طرفية حقيقية. الاختبارات تثبت حساب الأسطر لتدفقات معروفة؛ الطرفية الحقيقية تكشف سلوك المحاكيات حول cursor save/restore، الالتفاف، scrollback، وbracketed paste mode.
